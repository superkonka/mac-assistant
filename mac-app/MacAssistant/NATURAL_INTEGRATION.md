# Agents 和 Skills 自然融入会话设计

## 核心设计理念

**无需切换界面，一切在对话中完成。**

## 使用方式

### 1. @Agent 提及 - 即时切换

输入 `@Agent名称` 快速切换到指定 Agent：

```
用户: @GPT-4V 分析这张截图
系统: 🔄 通过 @GPT-4V 指定，已切换到 👁️ GPT-4V Vision

用户: [截图已发送]
GPT-4V: 这张截图显示的是...
```

**触发提示：** 输入 `@` 显示下拉建议列表

```
输入: @G
下拉: [@GPT-4V Vision, @Claude Code, @Kimi Local]
```

### 2. /Skill 命令 - 一键执行

输入 `/Skill名称` 直接触发技能：

```
用户: /screenshot 分析一下我的桌面
系统: ✅ 截图分析 执行成功
GPT-4V: 您的桌面显示有...
```

**触发提示：** 输入 `/` 显示可用技能列表

```
输入: /s
下拉: [/screenshot 截图分析, /summarize 总结文本]
```

### 3. 自然语言触发 - 智能检测

无需特殊语法，自然表达即可触发：

```
用户: 截个图看看我的桌面
系统: 💡 检测到 截图分析 意图
      
      "看看我的桌面"
      
      是否使用 📸 截图分析？
      回复 "是" 或 "y" 确认执行...

用户: 是
系统: ✅ 截图分析 执行成功
      [图片]
      GPT-4V: 您的桌面显示有以下内容...
```

### 4. 智能 Agent 建议 - 自动推荐

当检测到需要特定能力时，主动建议切换：

```
用户: 分析一下这段代码的问题
系统: 💡 当前 Agent 不支持代码分析
      
      建议切换到 📝 Claude Code 来处理此请求。
      
      是否切换？
      回复 "是" 或 "y" 确认切换并发送消息。

用户: 是
系统: 🔄 已切换到 📝 Claude Code
Claude: 这段代码存在以下问题...
```

## 输入框设计

```
┌──────────────────────────────────────────────────────────┐
│ 🤖 Kimi Local                                    ✨  📸  │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  User: 分析一下这段代码                               │
│                                                          │
│  💡 建议切换到 Claude Code 来处理此请求                 │
│      是否切换？回复 "是" 或 "否"                        │
│                                                          │
├──────────────────────────────────────────────────────────┤
│ 🌙 Kimi Local · kimi-k2.5 · [文本对话][代码分析]        │
├──────────────────────────────────────────────────────────┤
│ [⚡] [📷] 输入 @ 选择 Agent，/ 使用 Skill...  [⬆️]     │
└──────────────────────────────────────────────────────────┘
```

输入框提示：
- 空状态显示提示文字
- 输入 `@` 显示 Agent 列表
- 输入 `/` 显示 Skill 列表
- 智能检测意图时显示确认提示

## 技术实现

### 核心组件

| 组件 | 职责 |
|------|------|
| `ConversationIntelligence` | 解析输入中的 @ 和 / 引用 |
| `IntelligentInputView` | 智能输入框 + 下拉建议 |
| `CommandRunner` | 处理解析后的输入和确认 |
| `AISkillRegistry` | 管理可用技能 |

### 解析流程

```
用户输入
    ↓
[ConversationIntelligence.analyzeInput]
    ↓
检测 @Agent → 提取 AgentMention
检测 /Skill → 提取 SkillCommand
自然语言 → 检测 Skill 意图
         → 检测 Agent 需求
    ↓
[CommandRunner.processInput]
    ↓
处理 Agent 切换
执行 Skill 命令
或请求用户确认
或发送纯净文本
```

### 确认机制

对于检测到的意图（非显式命令），系统会：

1. **显示确认消息** - 说明检测到的意图
2. **等待用户回复** - "是"/"y" 确认，其他忽略
3. **使用 metadata 标记** - 标记待处理的意图
4. **下次输入处理** - 根据回复执行或取消

## 优势

1. **零学习成本** - 像平时聊天一样使用
2. **渐进式发现** - 从自然语言开始，逐步学习语法
3. **上下文保持** - 整个会话流程在一个界面完成
4. **智能提示** - 主动建议，减少用户思考
5. **灵活组合** - @Agent + /Skill + 自然语言随意组合
