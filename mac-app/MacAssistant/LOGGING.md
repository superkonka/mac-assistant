# Mac Assistant 日志系统

## 概述

v4.0.0 版本引入了完整的文件日志系统，所有运行日志会写入文件，方便调试和分析问题。

## 日志文件位置

```
~/Documents/MacAssistant/Logs/
├── mac-assistant-current.log    # 当前日志文件
└── Archive/                     # 归档日志目录
    ├── mac-assistant-2024-01-15-103045.log
    └── mac-assistant-2024-01-15-094530.log
```

## 日志级别

| 级别 | 图标 | 用途 |
|------|------|------|
| DEBUG | 🔍 | 详细调试信息 |
| INFO | ℹ️ | 一般信息 |
| WARNING | ⚠️ | 警告信息 |
| ERROR | ❌ | 错误信息 |
| CRITICAL | 🚨 | 严重错误 |

## 查看日志

### 方式一：应用内日志查看器

1. 点击状态栏图标
2. 选择 "📋 查看日志"（或按 ⌘+L）
3. 使用搜索、过滤功能查找问题

功能特点：
- 实时自动刷新
- 按级别过滤
- 关键词搜索
- 统计概览
- 一键导出

### 方式二：打开日志目录

1. 点击状态栏图标
2. 选择 "📂 打开日志目录"
3. 用任意文本编辑器打开日志文件

## 日志格式

```
[2024-01-15 10:30:45] [INFO] [main] File.swift:25 functionName - 日志消息
```

格式说明：
- `[时间]` - 精确到秒的日志时间
- `[级别]` - 日志级别
- `[线程]` - main 或 bg（后台线程）
- `文件名:行号 函数名` - 代码位置
- `-` 日志消息内容

## 代码中使用日志

```swift
// 在需要记录日志的文件中，直接使用全局函数：

LogDebug("调试信息")                                    // DEBUG 级别
LogInfo("一般信息")                                     // INFO 级别
LogWarning("警告信息")                                  // WARNING 级别
LogError("错误信息")                                    // ERROR 级别
LogError("操作失败", error: someError)                  // 带错误详情
LogCritical("严重错误")                                 // CRITICAL 级别
```

### 在类中使用

```swift
import Foundation

class MyClass {
    func doSomething() {
        LogInfo("开始执行操作")
        
        do {
            try riskyOperation()
            LogInfo("操作成功")
        } catch {
            LogError("操作失败", error: error)
        }
    }
}
```

## 日志轮转

- 单个日志文件最大 10MB
- 超过时自动归档，保留最近 30 天
- 历史日志自动清理

## 诊断问题流程

1. **用户发现问题** → 打开日志查看器或导出日志
2. **导出日志** → 日志查看器中点击 "💾 导出" 按钮
3. **发送日志文件** → 将导出的文件发送给开发者
4. **开发者分析** → 查看日志定位问题

## 隐私说明

日志中可能包含：
- ✅ 系统状态（CPU、内存、磁盘使用率）
- ✅ 应用名称（前台应用、活跃应用列表）
- ✅ 错误信息和堆栈跟踪
- ❌ 不会记录：密码、私人文本内容、个人文件内容

## 故障排除

### 日志文件未生成

1. 检查 Documents/MacAssistant/Logs 目录权限
2. 重启应用
3. 检查磁盘空间

### 日志查看器无法打开

1. 确保应用有文件访问权限
2. 尝试直接打开日志目录
3. 检查日志文件是否存在

## 技术实现

- **Logger.swift**: 核心日志实现
- **LogViewerView.swift**: 日志查看器 UI
- 使用 GCD 队列保证线程安全
- 批量写入优化性能
- 自动 flush 防止数据丢失
